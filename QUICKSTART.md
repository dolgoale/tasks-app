# Быстрый старт - Tasks

## Локальный запуск для тестирования

### Шаг 1: Запуск проекта

```bash
cd ~/Yandex.Disk.localized/Личное/Git\ Projects/Tasks
./start_dev.sh
```

Или вручную:

```bash
docker-compose -f docker-compose.dev.yml up --build
```

### Шаг 2: Откройте в браузере

- **Frontend (основное приложение)**: http://localhost:3001
- **Backend API документация**: http://localhost:8001/docs
- **Backend API**: http://localhost:8001

### Шаг 3: Тестирование

1. **Создание задачи:**
   - Нажмите кнопку "Создать задачу"
   - Заполните название (обязательно)
   - Опционально: описание, категория, приоритет, статус
   - Нажмите "Создать"

2. **Создание подзадачи:**
   - Создайте задачу как обычно
   - В поле "Родительская задача" выберите существующую задачу
   - Подзадача будет визуально отображена с отступом под родительской

3. **Редактирование задачи:**
   - Нажмите иконку редактирования (карандаш) рядом с задачей
   - Измените нужные поля
   - Нажмите "Сохранить"

4. **Отметка как завершенной:**
   - Установите галочку в колонке "Выполнено"
   - Или используйте кнопку редактирования и измените статус

5. **Фильтрация:**
   - Используйте переключатели вверху страницы:
     - **Все** - показать все задачи
     - **Не завершенные** - только активные задачи
     - **Завершенные** - только завершенные задачи

6. **Удаление задачи:**
   - Нажмите иконку удаления (корзина)
   - Подтвердите удаление
   - ⚠️ Задачи с подзадачами нельзя удалить (сначала удалите подзадачи)

### Остановка

Нажмите `Ctrl+C` в терминале или выполните:

```bash
docker-compose -f docker-compose.dev.yml down
```

### Просмотр логов

```bash
# Backend логи
docker-compose -f docker-compose.dev.yml logs -f backend

# Frontend логи
docker-compose -f docker-compose.dev.yml logs -f frontend
```

### Сброс базы данных

```bash
rm backend/data/tasks.db
```

База данных будет автоматически создана при следующем запуске.

## Структура данных

### Задача содержит:
- **Название** (обязательно)
- **Описание** (опционально)
- **Категория** (опционально)
- **Приоритет**: низкий, средний, высокий
- **Статус**: сделано, в работе, завершено
- **Завершенность**: да/нет

### Подзадачи:
- Могут иметь только одну родительскую задачу
- Визуально отображаются с отступом
- Наследуют структуру основной задачи

## Подготовка к развертыванию на tasks.aadolgov.com

После тестирования локально, для развертывания потребуется:

1. Создать production docker-compose файл
2. Настроить nginx для frontend
3. Настроить домен и SSL сертификаты
4. Настроить production базу данных (PostgreSQL)
5. Настроить переменные окружения для production
